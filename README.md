# DataScienceLearn

A privacy-first, browser-native Data Science learning app powered by **Chrome's Built-in AI (Gemini Nano)**. It generates mock tests and slide-based topic explanations entirely on-device — no servers, no API keys, no data ever leaves the browser.

Built with **React 19 · TypeScript 6 · Vite 8 · Tailwind CSS v4 · shadcn/ui**.

---

## Core Features

### Mock Test Engine

- **4 question types** per quiz: Conceptual, Math (with LaTeX), Programming (with code), Case Study
- **Both answer modes**: Single-select and Multi-select (all-or-nothing scoring)
- **4 experience levels** with distinct question distributions and time limits:

| Level | Duration | Questions | Conceptual | Math | Programming | Case Study |
|---|---|---|---|---|---|---|
| Beginner | 7 min | 10 | ✓ | — | — | — |
| Intermediate | 15 min | 15 | ✓ | ✓ | ✓ | — |
| Advanced | 20 min | 20 | ✓ | ✓ | ✓ | ✓ |
| Expert | 20 min | 25 | ✓ | ✓ | ✓ | ✓ |

- **Topic validation** — AI verifies the topic belongs to the Data Science domain before quiz generation
- **Live generation progress** — real-time stream preview with partial question count
- **Session persistence** — quiz state saved to `sessionStorage` keyed by UUID; resumable on page refresh
- **Countdown timer** with auto-submit and low-time warning styling
- **LaTeX rendering** — KaTeX-powered `MathText` component renders inline/block math in questions and options
- **Tab-switch tracking** — counts how many times the user left the tab during the quiz
- **Per-question timings** — seconds spent on each question recorded on navigation and submit

### Results Page

- **Hero card** — overlapping bar chart + donut ring visualization:
  - **Bar chart**: semi-transparent bars showing per-question time spent; clicking a bar opens the question's explanation
  - **Donut ring**: SVG arc chart showing correct / incorrect / skipped proportions
  - **Badges**: Correct count, Incorrect count, Skipped count, elapsed time ("Finished early"), tab switches
- **Full-page AI explanation** — clicking any bar navigates to `?explain=Q{n}`, a dedicated explanation page showing:
  - Question text with LaTeX rendering
  - Options in a plain annotated list (✓ correct, ✗ wrong picks)
  - Single-select / Multi-select badge
  - Streamed explanation generated by a dedicated AI session
  - Error state if the AI agent is not ready
  - Action buttons after generation: Prev Question, Next Question, Results, Try Again
- **Download** — exports all questions + cached explanations as a Markdown file
- **Try Again / Explain Topic** — quick actions to restart or deep-dive into the topic

### Explain Topic Engine

- **Slide-based learning** — AI decomposes any Data Science topic into 6–12 subtopics, then generates each slide on-demand
- **Lazy generation** — slides created one at a time as the user navigates (cached once generated)
- **Rich content** — markdown, inline code, LaTeX math notation, ASCII diagrams
- **Post-slide Q&A** — after all slides, a chat interface opens for follow-up questions with full topic context

## Routing

Uses `HashRouter` (works without a server / from `file://`).

| Route | Description |
|---|---|
| `/#/` | Landing page |
| `/#/mock-test` | Topic input + level selector |
| `/#/mock-test/:id/generating` | Live AI generation progress |
| `/#/mock-test/:id/quiz` | Active quiz with sidebar navigator + timer |
| `/#/mock-test/:id/results` | Score ring, bar chart, session analytics |
| `/#/mock-test/:id/results?explain=Q{n}` | Full-page explanation for question N |
| `/#/explain` | Explain topic — setup → slides → chat |

The `:id` is a `crypto.randomUUID()` generated at test setup. Setup data (`topic`, `level`) and quiz state (`questions`, `answers`, `seen`, `currentQ`) are persisted in `sessionStorage` under that UUID — this means the quiz survives page refresh and back-navigation.

`MockTestProvider` is the layout route for all `/mock-test/:id/*` routes, keeping AI agents and state alive across the generating → quiz → results transition.

## LLM Architecture

Three specialized agents share two Chrome AI sessions:

```
┌──────────────────────────────────────────────────────────────┐
│                     Chrome Built-in AI                       │
│                 (window.LanguageModel API)                    │
├─────────────────────────────┬────────────────────────────────┤
│       Quiz Session          │      Explanation Session        │
│   temp=1.5  topK=40         │    temp=0.8  topK=40           │
│   "Output only valid JSON"  │    "World-class DS educator"   │
│                             │                                │
│   PlannerAgent              │   ExplainerAgent               │
│   • validateTopic()         │   • explainQuestion()          │
│     (responseConstraint:    │     (per-question, streamed)   │
│      boolean schema)        │   • generateSlide()            │
│                             │   • chat()                     │
│   QuestionGeneratorAgent    │                                │
│   • generateQuestions()     │                                │
│     (responseConstraint:    │                                │
│      MCQQuestion[] schema)  │                                │
└─────────────────────────────┴────────────────────────────────┘
```

### Why two sessions?

The quiz session's system prompt forces JSON output. Reusing it for explanations makes the model output JSON-formatted text instead of natural language. A separate educator-focused session solves this cleanly.

### Agent Files

| File | Purpose |
|---|---|
| `src/agents/BaseAgent.ts` | Base class — session init, clone, destroy, `isReady` |
| `src/agents/promptApi.ts` | Chrome Prompt API wrapper — streaming, abort, init lifecycle |
| `src/agents/PlannerAgent.ts` | Topic validation, slide outline generation |
| `src/agents/QuestionGeneratorAgent.ts` | MCQ generation with JSON schema constraint + LaTeX fix |
| `src/agents/ExplainerAgent.ts` | Question explanations, slide content, chat |
| `src/agents/schemas.ts` | Zod schemas for question JSON validation |
| `src/lib/markdownRenderer.ts` | Markdown → HTML (headings, lists, code, math blocks) |
| `src/components/MathText.tsx` | KaTeX component — splits `$...$` / `$$...$$` and renders inline/block math |

### LaTeX Handling

The AI generates LaTeX inside JSON strings. `JSON.parse` corrupts bare `\frac`, `\nabla`, `\beta` etc. (`\f` → form-feed). Fixed in `QuestionGeneratorAgent.ts`:

```ts
// Only fix unescaped \X — negative lookbehind avoids corrupting already-correct \\frac
text.replace(/(?<!\\)\\([bfnrt])(?=[a-zA-Z])/g, '\\\\$1');
```

Parsed questions are rendered with `<MathText>` (quiz) and `renderSlideMarkdown()` (explanations/slides).

### Key Types

```typescript
type AIStatus = 'checking' | 'downloading' | 'ready' | 'unavailable' | 'unsupported' | 'error';
type AnswerValue = number | number[] | null;

interface MCQQuestion {
  id: number;
  questionType: 'conceptual' | 'math' | 'programming' | 'case-study';
  answerMode: 'single' | 'multiple';
  question: string;
  options: Array<{ content: string; optionType: 'text' | 'math' | 'code' | 'case-study' }>;
  correct: number | number[];
  code?: string;
  context?: string;
  diagram?: string;
}
```

## Styling Architecture

- **No `tailwind.config.js`** — all theme tokens defined in `src/index.css` via `@theme inline {}`

## Session Persistence

Quiz state is persisted to `sessionStorage` so the user can refresh or navigate back without losing progress.

| Key | Contents |
|---|---|
| `ds_mocktest_setup_<uuid>` | `{ topic, level }` — written at setup |
| `ds_mocktest_state_<uuid>` | `{ questions, answers, seen, currentQ }` — written on every state change |

State is cleared on **Try Again** or **Explain Topic**. It is NOT cleared on page refresh, allowing resume.

## Assumptions & Constraints

1. **Chrome 138+ required** — Chrome's `window.LanguageModel` Prompt API is a Chrome-only feature behind a flag.
2. **Gemini Nano limitations** — The on-device model has a limited context window. For 25-question (Expert) quizzes, the parser uses a brace-counting fallback to extract partial results if JSON is truncated.
3. **Single-file build** — Production output is one `dist/index.html` (~700 KB). Enables offline use and easy sharing; no code splitting.
4. **Multi-select scoring** — All-or-nothing; partial selection scores zero.
5. **Explanation caching** — Each explanation is generated once per results session and cached in a `useRef`. Navigating away does not clear the cache (it lives in the context).
6. **Tab-switch count** — Tracks `document.visibilitychange` events during the quiz only. Uses `useRef` to avoid Strict Mode double-counting.

## Development Notes

### Adding a shadcn/ui Component
```bash
pnpm dlx shadcn@latest add <component-name>
```

### Adding a New Question Type
1. Add to `MCQQuestion['questionType']` in `src/lib/config.ts`
2. Add distribution range to each level in `LEVEL_CONFIG`
3. Update `buildMockTestPrompt()` in `QuestionGeneratorAgent.ts`
4. Add rendering in `TestQuiz.tsx`
